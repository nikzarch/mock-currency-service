
Mock Currency Service
=====================

Описание
--------
Mock Currency Service — это небольшой HTTP‑сервис на Go, который имитирует ответ
эндпоинта курса валют:

    /scripts/XML_daily.asp?date_req=02/03/2002

Он генерирует детерминированные XML‑отчёты по курсам валют на основе даты
запроса и сохраняет их в PostgreSQL, чтобы повторные запросы возвращали
одинаковые данные.

Сервис разработан для параллельного запуска большого количества запросов.
Отчёты кэшируются в базе данных по дате, поэтому одинаковые запросы
возвращают идентичные ответы.

При первом запросе для конкретной даты отчёт генерируется и сохраняется,
после чего используется повторно.

Для проверки сценариев отказа можно запустить сервис с переменной:

    RESPONSE_MODE=error

В этом режиме сервис будет возвращать HTTP 500 на все запросы.

Основные возможности
--------------------

- HTTP‑совместимый endpoint `/scripts/XML_daily.asp`
- генерация XML‑отчёта по дате
- хранение отчётов в PostgreSQL
- безопасная работа при большом количестве параллельных запросов
- режим принудительной ошибки для тестирования отказов
- health‑endpoint `/healthz`
- запуск базы данных через Docker Compose

Архитектура
-----------

Проект организован по принципу доменных пакетов:

    cmd/                  точка входа приложения
    internal/app          сборка и запуск HTTP‑сервера
    internal/config       конфигурация
    internal/db           подключение к PostgreSQL
    internal/health       health‑check handler
    internal/valute       доменная логика валютного отчёта
    migration/            SQL‑миграции для базы данных
    api/proto             protobuf контракт
    api/swagger           OpenAPI описание

Поток обработки запроса:

HTTP request
→ Handler
→ Service
→ Repository
→ PostgreSQL

При отсутствии отчёта в базе сервис генерирует его и сохраняет,
после чего возвращает XML клиенту.

Endpoint
--------

Получение ежедневного отчёта:

    GET /scripts/XML_daily.asp?date_req=02/03/2002

Параметры:

- date_req — дата отчёта в формате DD/MM/YYYY

Пример запроса:

    curl "http://localhost:8080/scripts/XML_daily.asp?date_req=02/03/2002"

Health check:

    GET /healthz

Запуск проекта
--------------

Требования:

- Go 1.24+
- Docker
- Docker Compose

1. Запуск PostgreSQL:

   make up

2. Запуск сервиса:

   make run

После запуска сервис будет доступен по адресу:

    http://localhost:8080

Переменные окружения
--------------------

HTTP_PORT
адрес HTTP сервера (по умолчанию :8080)

DB_POSTGRES_URL
строка подключения к PostgreSQL

DEFAULT_DATE_REQ
дата по умолчанию, если параметр date_req не передан

RESPONSE_MODE
режим работы сервиса:
success — нормальная работа
error   — всегда возвращается HTTP 500

HTTP_READ_TIMEOUT
HTTP_WRITE_TIMEOUT
HTTP_IDLE_TIMEOUT
таймауты HTTP сервера
